# Contributing to the CESARops Ecosystem

Thank you for your interest in contributing to the CESARops ecosystem! We are building a suite of tools for Search and Rescue teams, and we welcome all the help we can get.

## Our Mission

Our core mission is to provide professional-grade, open-source tools to SAR teams for free. This includes the [CESARops platform](https://cesarops.com/), [Sonar Sniffer](https://cesarops.com/), and other related libraries like this one. By contributing, you are helping to build software that can make a real-world difference.

## How to Contribute

There are many ways to contribute to the project, and you don't have to be a developer to help.

*   **Developers:** We need help with everything from bug fixes to new features. If you are a Rust, C++, or web developer, we have a place for you. Key areas include signal processing, UI development, and backend services.
*   **Designers:** We are always looking for help with UI/UX design to make our tools more intuitive and easier to use in high-stress situations.
*   **SAR Professionals:** Your feedback is invaluable. We need your expertise to ensure we are building the right tools for the job. You can help by testing our software, providing feedback, and sharing your experiences.
*   **Technical Writers:** Help us improve our documentation, write tutorials, and make our projects more accessible to everyone. Clear documentation is critical for adoption.

## Getting Started

To get started, please visit our main organization page on GitHub (once you create it). There you will find our main issue tracker, contribution guidelines, and a roadmap for the project.

We look forward to collaborating with you and building something great for the SAR community.
